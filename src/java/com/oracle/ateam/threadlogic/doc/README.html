ThreadLogic (version 0.95)

Date: March 1, 2012

Note: This is the version 0.95 release of the software This is a pre-release version.

Changes since 0.9

  • Cache the thread data instead of reparsing thread repeatedly so that sorting does not suffer for names/advisories
  • Clean up Thread Dump & ThreadGroup overview page display
  • Excluding [ACTIVE]/[STUCK] labels during by thread names
  • Support parsing of partial thread name labels for Sun & JRockit Parsers.
  • Cleaned up sorting based on thread name, thread ids, nids.
  • Reset Min Default filter level to IGNORE as users are forgetting about it and report not seeing threads although the numbers are listed.
  • Fixed search by thread name for various models.
  • Reduced diagnostic logging during thread parsing.
  • Added support for external reference of Advisory & Group Definitions files
  • Use -Dthreadlogic.advisories=directory... command line argument to pick custom advisories from a named directory (in addition to packaged advisories).
  • Use -Dthreadlogic.groups=directory... command line argument to pick customized group definitions from a named directory (instead of predefined groups).
  • Support for reporting thread group data as a column within thread model as well as in merged view.
  • Fixed wrong tagging of parked threads as blocked threads.
  • Added OSB related advisories & enhanced OSB thread grouping
  • Optimized searches for timestamp
  • Enhanced the various (JRockit/IBM, enhancing Hotspot) parsers to report the timestamp and jvm version within thread dumps and also in the Merged view to report the timestamp during thread comparison.
  • Remove the JconsolePlugin service file
  • Cleanup some html output. Only need one set of html/body tags in the htmlPane.
  • Fixed: Advisory should be a tooltip popup
  • Fixed: Saving/loading session.
  • Added support for multi-line pattern for advisories as well as documented the usage inside AdvisoryMap.xml.
  • Included pure coherence threads into Top WLS grouping which got missed out earlier.
  • Updated HotCallPattern & Filter to ignore lock ids which will break pattern matching for threads displaying same code execution and added additional Advisories for OSB and updated other advisory entries...
  • Added advisories for OSB, Oracle DMS, MBean server queries
  • Excluded AQ adapter threads from Oracle Framework thread group
